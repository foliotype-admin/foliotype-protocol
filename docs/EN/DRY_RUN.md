## 1. Protocol Concept
The **Dry Run** is a mandatory simulation step before any final export. It serves to validate the integrity of the processing chain without committing final storage resources.

## 2. Control Points (Checklist)
Before validating a session, the following elements are fine-combed:
* **Routing:** Verification that no signal saturates internal buses.
* **Automations:** Full playback to detect any artifacts or gain jumps.
* **CPU Load:** System stability during real-time rendering.
