## 1. Scope
This directory centralizes dynamic resources, demonstrations, and broadcast-ready media. It serves as the primary rendering library for the **Foliotype Protocol**.

## 2. Directory Structure
The organization is sectorized by media type to optimize access and deployment:

* **Videos/** : Demo captures, cinematics, and animated banners (.mp4).
* **Images/** : Scene renders, system captures, and UI assets (.png, .jpg).
* **Demos/** : Real-world applications of the design system.

## 3. Management & Optimization Protocol
1. **Web Performance**: Every `.mp4` file must be compressed using the H.264 codec before integration to ensure seamless playback.
2. **Naming Convention**: Strict application of `slug-case` (e.g., `demo-interface-v1.mp4`).
3. **Source Separation**: Editable project files (After Effects, etc.) are prohibited here; they reside exclusively in the `masters/` root folder.

---
## 4. Maintenance & Integrity
* **Sync Threshold**: Recommended maximum weight of 50 MB per file.
* **Validation**: All new assets must be tested on the `preflight` branch.
